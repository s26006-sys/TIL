# 객체 지향이란?
객체 지향이란 프로그래밍에서 필요한 데이터를 추상화 시켜 ` 상태와 행위를 가진 객체 `로 만들고, 이들의 상호작용으로 데이터를 처리하는 설계 방식

* 여기서 객체란?<BR>
  객체는 프로그램에서 사용되는 데이터 또는 식별자에 의해 참조되는 공간<BR>
  값을 저장 할 `변수`와 작업을 수행 할 `메소드`를 서로 연관된 것들끼리 묶어서 만든 것을 객체라고 할 수 있다.
  <BR><BR><BR>


# 객체 지향 프로그램의 특징
  1.추상화<BR>
  2.캡슐화<BR>
  3.상속<BR>
  4.다형성<BR>
<BR>
## 추상화
- 객체에서 공통된 속성과 행위를 추출 하는 것
- 공통의 속성과 행위를 찾아서 타입을 정의하는 과정
- 추상화는 불필요한 정보는 숨기고 중요한 정보만을 표현함으로써 프로그램을 간단하게 만드는 것

## 캡슐화
- 데이터 구조와 데이터를 다루는 방법들을 결합 시켜 묶는 것 (변수와 함수를 하나로 묶는 것을 뜻함)
- 낮은 결합도를 유지할 수 있도록 설계하는 것
* 캡슐화를 통해서 정보은닉을 활용 할 수도있다.

## 상속
- 클래스의 속성과 행위를 하위 클래스에 물려주거나 하위 클래스가 상위 클래스의 속성과 행위를 물려받는 것을 말한다
- 새로운 클래스가 기존의 클래스의 데이터와 연산을 이용할 수 있게 하는 기능

# 객체 지향 프로그래밍의 장단점

## 장점
- 클래스 단위로 모듈화시켜서 개발하기 때문에 업무 분담이 편리하고 대규모 소프트웨어 개발에 적합하다.
- 클래스 단위로 수정이 가능하기 때문에 유지 보수가 편리하다.
- 클래스를 재사용하거나 상속을 통해 확장함으로써 코드 재사용이 용이하다.

## 단점
- 처리속도가 상대적으로 느리다.
- 객체의 수가 많아짐에 따라 용량이 커질 수 있다.
- 설계시 많은 시간과 노력이 필요하게 될 수 있다.

코드 예시
```
# 1. 클래스 정의 (설계도)
class Animal:
    def __init__(self, name):
        self.name = name  # 속성 (데이터)

    def speak(self):      # 메서드 (동작)
        pass

# 2. 상속 (Animal의 기능을 물려받아 확장)
class Dog(Animal):
    def speak(self):
        return f"{self.name}: 멍멍!"

class Cat(Animal):
    def speak(self):
        return f"{self.name}: 야옹~"

# 3. 객체 생성 (실제 인스턴스 만들기)
my_dog = Dog("초코")
my_cat = Cat("나비")

print(my_dog.speak())  # 출력: 초코: 멍멍!
print(my_cat.speak())  # 출력: 나비: 야옹~
```
