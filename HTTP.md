# HTTP란?
HTTP는 웹 브라우저(클라이언트)와 웹 서버가 HTML, 이미지 등 데이터를 주고받기 위한 통신 규약이다.

## HTTP 장점과 단점

### 장점
 - 단순하고 이해하기 쉽다
 - 확장성과 유연
 - 플랫폼 독립적이다
 - 전 세계적으로 표준화된 프로토콜

### 단점
 - 보안이 취약하다
 - 연결 방식이 비효율적이다
 - 실시간 통에 불리하다

# HTTP Method란?
HTTP 메서드는 클라이언트(브라우저)가 서버에 보낼 때 ` 내가 지금 어떤 의도로 이 요청을 하는 거야 `라고 알려주는 행동 지침이다.

## 대표적인 HTTP 메서드
1.GET(조회)
 - 서버에서 데이터를 가져올 때

2.POST(생성)
 - 새로운 리소스를 생성하거나 데이터를 서버로 제출할 때 사용

3.PUT(수정)
 - 리소스의 전체를 교체(수정)할 때 사용 (존재하지 않으면 생성)

4.PATCH(일부 수정)
 -  리소스의 부분적인 내용을 수정할 때 사용

5.DELETE(삭제)
  - 리소스를 삭제할 때 사용

# HTTP Status code란?
HTTP Status code는 클라이언트(브라우저 등)가 보낸 요청의 처리 결과를 서버가 숫자로 응답하는 3자리 코드이다

자주 나오는 상태 코드
- 2xx (성공)
  - 200 OK : 요청이 성공
  - 201 Created : 요청이 성공하여 새로운 리소스가 생성됨
- 3xx (리다이렉션)
  - 301 Moved Permanently: 요청한 리소스의 URI가 영구적으로 변경됨
  - 302 Found: 요청한 리소스가 일시적으로 다른 URI에 존재함
- 4xx (클라이언트 에러)
  - 400 Bad Request: 잘못된 문법 등으로 서버가 요청을 이해하지 못함
  - 401 Unauthorized: 인증(로그인 등)이 필요함

- 5xx (서버 에러)
   - 500 Internal Server Error: 서버 내부 오류 (코드 에러 등)
   - 503 Service Unavailable: 서버 점검이나 과부하로 서비스 이용 불가
  
# HTTP의 무상태성
HTTP의 무상태성은 서버가 클라이언트의 이전 요청이나 상태를 기억하지 않는 특성이다.

# HTTP와 HTTPS의 차이점
 - 보안 -> HTTP는 암호화를 하지 않는다 , 그러나 HTTPS는 데이터 암호화 하여 안전하다
